## Key Technical Findings

### 1. max_tokens is Critical for Eval
- Qwen3-8B needs 32K+ tokens for AIME (official recommends 38912)
- SFT models (from Base) have `max_position_embeddings=32768`, use `max_tokens=30000`
- Post-trained models have `max_position_embeddings=40960`, use `max_tokens=32768`
- Previous evals with max_tokens=8192 severely underestimated all models

### 2. OPD Training Settings
- `max_response_len=8192` caused 100% truncation → model never sees complete reasoning
- Should use `max_response_len=16384` minimum for next experiments
- Current OPD config: `scripts/train-opd.sh` + `configs/opd-cumulative.env`

### 3. Teacher Consistency (Lightning OPD)
- SFT data MUST be generated by the same teacher used for OPD
- Our current SFT uses OpenThoughts3 (QwQ-32B) → teacher inconsistency with Qwen3-8B OPD
- Need to generate SFT data using Qwen3-8B itself
- Lightning OPD pipeline: extract prompts → teacher generates responses → SFT → OPD

### 4. Cumulative vs Instant OPD
- Cumulative OPD has much more stable training (grad_norm 0.16 vs 3.0)
- But instant OPD achieved better eval results (+10pt vs +3pt)
- Possible reason: cumulative kl_coef=0.05 was too small (magnitude ~8x larger due to horizon sum)
- Need to tune: try cumulative with kl_coef=0.01-0.02, or instant with kl_coef=0.5

### 5. SGLang Server Issues
- `--max-total-tokens` must not exceed model's `max_position_embeddings`
- Use `health_generate` endpoint (not `health`) to check readiness
- Old server processes can linger and cause port conflicts → always `pkill -9 sglang` before starting
- Container restart needed to clear zombie processes
